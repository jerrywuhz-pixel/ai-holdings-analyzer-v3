# 待确认问题

这些是我认为不能擅自决定的问题。确认后才能进入 PRD、数据库变更和实现阶段。

## 最高优先级

1. **已确认：账号 = 持仓 3.0 系统账号。**  
   用户通过邮箱、账号 ID 或手机号登录；绑定微信 clawbot 插件后获得渠道消息交互能力。账号下可通过手工录入、买入/卖出消息、券商 skills 直连 API 等方式获取资产数据，并支持整合多券商账户形成统一资产视图。所有后台写入必须记录来源。

2. **一个微信 claw bot 是否可以绑定多个系统账号？**  
   当前建议默认一对一：一个系统账号绑定一个主要微信 clawbot。若允许一对多，需要定义微信内切换和误推送保护。

3. **3.0 的基座是否接受“OpenClaw Gateway + 独立 Environment Orchestrator”的拆分？**  
   还是你希望明确选择 OpenClaw-first 或 Hermes-first？

4. **Hermes 的角色边界是什么？**  
   只做机会猎手/深研子 agent，还是承担日常对话、memory 和 cron？

5. **券商生产账号接入权限是只读，还是允许创建交易草稿，甚至后续自动交易？**  
   我建议 3.0 默认只读，同步真实持仓、成交、现金和保证金。

6. **第一批券商优先级怎么排？**  
   富途、长桥、PTrade 三者中谁是第一目标？是否有已经可用的 API 权限？

7. **期权模块第一阶段是否只做美股 sell put？**  
   港股期权、A 股 ETF 期权是否纳入 3.0 首版？

8. **Deep Research 的触发和预算边界是什么？**  
   哪些问题自动升 GPT-5.5？是否需要用户确认“这次会用深研模型”？

9. **MiniMax M2.7 的供应商入口是什么？**  
   官方 MiniMax、OpenRouter、NVIDIA NIM，还是统一模型网关？

10. **已确认：WebApp 不坚持业务只读。**  
    3.0 允许 WebApp 管理 `portfolio_view`、关注清单、清仓复盘、风险偏好、通知规则、交易纪律、手工交易草稿、确认会话和富途同步触发；所有写入仍必须走 Domain Tools、确认流、审计和幂等控制。

## 下一步建议

建议先确认以下 5 个问题，再进入 PRD：

1. 账号模型：`tenant_id/channel_binding_id/openclaw_account_id/asset_source_id/broker_connection_id` 已成立；`portfolio_view_id` 首期支持多个，用于视图和策略分组。
2. 基座架构：是否采用 OpenClaw Gateway + 独立 Environment Orchestrator。
3. 券商优先级和权限：富途/长桥/PTrade，默认只读还是更高权限。
4. 期权首版范围：是否只做美股 sell put。
5. 模型路由：MiniMax M2.7 日常 + GPT-5.5 深研是否接受，以及触发/预算规则。
