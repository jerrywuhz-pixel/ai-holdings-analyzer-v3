## Residual Risks

- The SWAS API profile did not list the host in any checked region, while DNS and SSH verified the live IP directly. Treat SSH/DNS evidence as the V1.0 source for this host.
- The local `scripts/hermes_swas_runbook.py` still assumes older SWAS CLI names and `127.0.0.1` data-service probing; the live server binds internal services to `172.17.0.1`.
- `wechat-clawbot-bridge` runs without a Docker healthcheck, so container `Up` is weaker evidence than the healthy status used by other services.
- User-visible WeChat delivery was not replayed in this maintenance pass; DB and domain-tool evidence prove runtime readiness, not a fresh WeChat round trip.
