# Trade Input Skill — 交易录入

## 触发条件

用户通过微信 clawbot 发送交易指令消息。触发关键词包括：
- 明确交易动词：买入、卖出、BUY、SELL、买、卖、加仓、减仓、清仓、平仓
- 隐含交易语境：持有、建仓、止盈、止损（结合数量和价格时）

**非触发条件**：仅查询持仓、行情咨询、闲聊消息。

## 输入格式

用户自然语言指令，支持以下维度解析：

| 维度 | 必填 | 示例 | 说明 |
|------|------|------|------|
| 方向 | 是 | 买入、卖出、BUY、SELL | 支持中英文 |
| 股票 | 是 | 茅台、600519、AAPL | 中文名称 / A股代码 / 美股代码 |
| 数量 | 是 | 100股、100 | 单位默认为股，省略时提取数字 |
| 价格 | 否 | @1800、成本价1800、1800元 | @前缀或"成本价"前缀 |
| 日期 | 否 | 今天、昨天、2024-01-15 | 默认今天（T+0） |
| 标签 | 否 | 突破确认、回调低吸、业绩驱动、趋势跟随、事件套利、仓位试单 | 多标签用逗号或空格分隔 |

**输入正则模式（参考）**：
```
(买入|卖出|BUY|SELL|买|卖|加仓|减仓)\s*(.+?)\s*(\d+)\s*股?\s*(?:[@成本价]?\s*(\d+(?:\.\d+)?))?
```

## 处理逻辑

1. **意图识别**：NLP 提取方向（side: BUY/SELL）
2. **股票解析**：
   - 纯数字（6位/5位）→ 直接作为 symbol
   - 中文名称 → 查询 `stock_catalog` 表映射到 symbol
   - 美股代码（字母）→ 直接作为 symbol，market='US'
   - A股 6 位数字默认后缀：SH（60/68开头）或 SZ（00/30开头）
3. **数量与价格提取**：正则 + NLP 双重提取
4. **日期解析**：
   - "今天" → 当前日期（YYYY-MM-DD）
   - "昨天" → 当前日期 - 1
   - 具体日期 → 校验格式并合法化（不可超过今天）
5. **标签标准化**：匹配预定义标签字典，未知标签提示用户
6. **用户确认**：生成确认消息，等待用户回复"确认"
7. **写入数据库**：用户确认后执行 INSERT

**状态机**：
```
IDLE → PARSING → AWAITING_CONFIRMATION → WRITING_DB → COMPLETED
                       ↓ 用户取消/超时
                  CANCELLED
```

## 输出格式

### 确认消息（给用户）
```
确认[方向] [股票名称]([market][symbol]) [数量]股 @[价格]，标签：[标签]。
回复"确认"后记录到持仓系统；不会自动下单。回复"取消"放弃。
```

### 成功响应
```
已记录[方向]交易。
[股票名称]({symbol}) {quantity}股 @{price} | {date} | 标签: {tags}
```

### 数据库写入字段
```json
{
  "id": "uuid",
  "user_id": "uuid",
  "symbol": "600519.SH",
  "side": "BUY",
  "quantity": 100,
  "price": 1680.00,
  "currency": "CNY",
  "trade_date": "2024-01-15",
  "tags": ["业绩驱动"],
  "source": "manual_chat",
  "raw_message": "买入茅台100股，成本价1680，标签业绩驱动",
  "created_at": "timestamp",
  "confirmed_at": "timestamp"
}
```

## 错误处理

| 错误场景 | 处理方式 |
|----------|----------|
| 股票名称无法识别 | 提示："未找到'XX'对应的股票，请提供代码（如600519）或检查名称" |
| 股票名称有多个匹配 | 列出候选："找到多个匹配：1. 贵州茅台 2. 茅台醇业。请回复数字选择" |
| 缺少方向或数量 | 追问："请补充交易方向（买入/卖出）和数量" |
| 价格缺失 | 追问："请补充成交价格（如@1680），或回复'市价'记录为 0" |
| 日期超过今天 | 拒绝："交易日期不能晚于今天" |
| 用户取消确认 | 状态置为 CANCELLED，不写入数据库 |
| 确认超时（5分钟） | 自动取消，提示"交易确认已超时，请重新发送" |
| 数据库写入失败 | 回滚状态，提示"记录失败，请稍后重试" |

## 示例

### 示例 1：完整中文指令
```
用户：买入茅台100股，成本价1680，标签业绩驱动
系统：确认买入 贵州茅台(SH600519) 100股 @¥1680.00，标签：业绩驱动。
      回复"确认"后记录到持仓系统；不会自动下单。回复"取消"放弃。
用户：确认
系统：已记录买入交易。
      贵州茅台(SH600519) 100股 @¥1680.00 | 2024-01-15 | 标签: 业绩驱动
```

### 示例 2：简写英文指令
```
用户：BUY AAPL 50 @190
系统：确认买入 Apple Inc.(US:AAPL) 50股 @$190.00。
      回复"确认"后记录到持仓系统；不会自动下单。
用户：确认
系统：已记录买入交易。
```

### 示例 3：指定日期
```
用户：昨天卖出宁德时代200股@210，趋势跟随
系统：确认卖出 宁德时代(SZ300750) 200股 @¥210.00，标签：趋势跟随。
用户：确认
系统：已记录卖出交易。
```

### 示例 4：股票名称歧义
```
用户：买入平安100股@50
系统：找到多个匹配：
      1. 中国平安(SH601318)
      2. 平安银行(SZ000001)
      请回复数字选择，或发送准确代码。
用户：1
系统：确认买入 中国平安(SH601318) 100股 @¥50.00...
```

## 数据库操作

- **trade_events**：INSERT（用户确认后写入，source='manual_chat'）
- **stock_catalog**：SELECT（股票名称 → symbol 映射查询）
- **users**：SELECT（通过微信 openid 获取 user_id）
