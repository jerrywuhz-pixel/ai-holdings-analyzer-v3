# Broker Parse Skill — 券商成交提醒解析

## 触发条件

用户通过微信 clawbot 转发券商成交提醒消息。识别特征：
- 消息来自券商公众号/服务号/企业微信（如"华泰证券"、"中信证券"、"招商证券"、"富途牛牛"）
- 消息内容包含成交确认关键词：成交、委托成交、已成、部分成交、买入成交、卖出成交
- 消息包含股票代码、价格、数量等结构化字段

**优先级**：Broker Parse Skill 优先级高于 Trade Input Skill。如果消息同时满足两者触发条件，优先按券商消息解析（更结构化，准确率更高）。

## 输入格式

券商成交提醒为微信消息文本，不同券商格式差异大。以下为典型格式模板：

```
成交价格：1680.00元
成交金额：168000.00元

委托编号：12345678
```

## 处理逻辑

1. **券商识别**：根据消息头部标识 `[券商名]` 或特定关键词匹配券商类型
2. **指纹生成**：对原始消息文本计算 MD5 作为 `broker_message_fingerprint`
3. **去重检查**：查询 `trade_events` 表，若 fingerprint 已存在则返回已存在提示
4. **字段提取**：使用券商专属正则模板提取：方向、symbol、名称、数量、价格、时间
5. **数据标准化**：
   - 方向统一映射：买入/BUY/买 → BUY；卖出/SELL/卖 → SELL
   - A股代码自动补全后缀（.SH / .SZ）
   - 美股代码保持原样，market='US'
   - 价格统一转为 Decimal
   - 时间解析为 timestamp，缺失日期用当天
6. **用户确认**（可选配置）：高价值交易（金额 > 100,000）可要求用户确认
7. **写入数据库**：INSERT trade_events，source='broker_wechat'

### 券商正则解析模板

```javascript
// 华泰证券
const HUATAI_REGEX = {
  name: /证券名称[：:]\s*(.+)/,
  symbol: /证券代码[：:]\s*(\d{6})/,
  side: /委托方向[：:]\s*(买入|卖出)/,
  quantity: /成交数量[：:]\s*(\d+)\s*股?/,
  price: /成交价格[：:]\s*(\d+(?:\.\d+)?)/,
  time: /成交时间[：:]\s*(\d{2}:\d{2}:\d{2})/,
  orderNo: /委托编号[：:]\s*(\d+)/
};

// 中信证券
const ZHONGXIN_REGEX = {
  side: /(买入|卖出)\s+(\d{6})/,
  symbol: /(买入|卖出)\s+(\d{6})/,
  name: /\d{6}\s+(.+)/,
  quantity: /成交量[：:]\s*(\d+)\s*股?/,
  price: /成交价[：:]\s*(\d+(?:\.\d+)?)/,
  time: /成交时间[：:]\s*(\d{4}-\d{2}-\d{2}\s+\d{2}:\d{2}:\d{2})/
};

// 招商证券
const ZHANSHANG_REGEX = {
  side: /成交方向[：:]\s*(买入|卖出)/,
  nameSymbol: /股票名称[：:]\s*(.+?)\((\d{6})\)/,
  quantity: /成交数量[：:]\s*(\d+)\s*股?/,
  price: /成交均价[：:]\s*(\d+(?:\.\d+)?)/,
  time: /成交时间[：:]\s*(\d{2}:\d{2}:\d{2})/
};

// 富途牛牛
const FUTU_REGEX = {
  side: /方向[：:]\s*(BUY|SELL)/i,
  symbol: /代码[：:]\s*([A-Z]+)/,
  name: /名称[：:]\s*(.+)/,
  quantity: /数量[：:]\s*(\d+)\s*股?/,
  price: /价格[：:]\s*\$?(\d+(?:\.\d+)?)/,
  time: /时间[：:]\s*(\d{4}-\d{2}-\d{2}\s+\d{2}:\d{2}:\d{2})/
};
```

## 输出格式

### 去重命中
```
该成交记录已存在（{fingerprint_short}），无需重复录入。
若需更新，请使用"修改交易 {id}"指令。
```

### 成功响应
```
已解析[{券商}]成交提醒：
[方向] [股票名称]({symbol}) {quantity}股 @{price}
成交时间：{datetime}
确认后记录到持仓系统；不会自动下单。
```

## 错误处理

| 错误场景 | 处理方式 |
|----------|----------|
| 无法识别券商格式 | 降级到 Trade Input Skill 处理，提示"未识别券商格式，尝试通用解析" |
| 正则提取字段缺失 | 追问用户补充缺失字段（symbol / 价格 / 数量） |
| fingerprint 已存在 | 提示"该成交记录已存在"，展示已有记录简要信息 |
| 代码格式异常 | 提示"股票代码格式异常：{code}，请确认是否为 A股/美股" |
| 时间与当天偏差 > 7天 | 警告"成交日期为 {date}，距今超过7天，请确认是否正确" |
| 数据库唯一约束冲突 | 捕获 PostgreSQL unique_violation，转为友好提示 |

## 数据库操作

- **trade_events**：INSERT（解析成功后写入，source='broker_wechat'）
- **trade_events**：SELECT（按 `broker_message_fingerprint` 去重查询）
- **users**：SELECT（通过微信 openid 获取 user_id）

## 扩展指南

新增券商支持步骤：
1. 收集该券商 3-5 条真实成交提醒样本
2. 编写专用正则模板，确保字段提取率 > 95%
3. 在 `broker_registry` 中注册新券商
4. 补充单元测试样本
5. 更新本文档"支持券商"列表
